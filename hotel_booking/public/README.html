EZ ROOMS        

                            User’s Manual Page                                                          PR02-15

As EZ Rooms website is under development we decided to write a user’s manual page, so it is easier for the user to navigate through the website until the final version is updated.

Contents:

         page1.html        - Homepage

         page2.html    - Hotel’s list page  

         page3.html    - Hotel information page

         page4.html    - Confirmation of booking ( and user information) page

         group.css      - CSS Stylesheet for all the pages above

         scripts.js        - js script file

         images(x15)

        

Main Features:

        Homepage  (page1.html)

  • User loads the website homepage, and has a wide range of options to search hotels in different cities of Australia via entering in the search bar, choosing one of the options from “our plans” box , or reading the reviews from review box.
  • User decides to enter the name of the location/destination/landmark and clicks on the search button symbol.

  • As our website is under development, and we still havn’t connected it to a database, clicking on the search button will direct it to our 2nd page, which gives list of hotels available with a google map.
  • The header and footer are constant on all the pages.
  • Clicking on the Ez Rooms logo takes you back to our homepage.

        Hotel’s list page ( page2.html)

  • Displays the images for different hotels on the google map provided on the left.
  • Has various features for selecting the number of beds, start date, end date and selecting number of stars.
  • Clicking on one of the hotel images takes you to our third page, verification of booking information/modification 

                             

     

                Information of Selected Hotel rooms page ( page3.html)

  • Displays the facilities available for the selected hotel room, amenities, places around to visit
  • Has options to change the search criteria

  • If you click on “Book” it will direct you to our 4th page, where user can VERIFY/MODIFY information

  • There is a review box on this page as well for the user’s who are still deciding whether the selected hotel room is their favoured room.

                Verification of booking/modification page (page4.html)

  •     Displays the information of the user ( name, email, contact, address)
  • Displays the selected hotel rent, stay date, number of rooms and the amount.
  •  In future, this page will redirect to login/sign up page, if the user is has not done either of it.
  •  Clicking on the proceed to payment will direct the user to payment portal, which is not the part of the milestone right now, so we directed it to our homepage.

                                             

  • Clicking on the Back button, allows the user to make any changes in the selection of rooms and facilities, even the hotel itself.

                        

Features in a nutshell:

  • User can book a room and manage the booking ( make modifications and confirm the booking)
  • The webpages are HTML, CSS, Js and Jquery encoded.
  • The HTML/CSS pages pass the validation test ( pag1.html , page2.html)
  • The js scripts passes the linting test.
  • The header and footer are consistent throughout the website and are interactive.
